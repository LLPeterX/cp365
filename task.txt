Меню:
  - Входящие
   - Обработка файла AFN из ФНС  -> ProcessAFN(null)
   - Получить из ПТК -> ProcessPTK()
  - Исходящие
   - Шифрование -> SendFiles()
   - Копировать в ПТК  -> CopyToPTK()
  - Настройка
  - Выход -> выгрузка сигнатуры, Application.Exit()

При запуске основной формы:
  - Загрузить Config из cp365.ini
    - Если файла нет, то создать его по умолчанию
    - Если ошибка, то выдать окно конфигурации, сохранить и перечитать
  - проверить наличие файлов arj32.exe, spki1utl.exe( или crypt32.exe),
     если VirualFDD - то файлов <profile>.img, imdisk.exe
  - Инициализировать СКАД "Сигнатура"
   - если VirtualFDD = 1, то запустить imdisk.exe и смонтировать <profile>.img
   - рассмотреть возможность исп-я crypt32.exe и spki1utl.exe 
     (две стратегии) - позже
   - инициализировать ДСЧ и загрузить ключи


 ProcessAFN(file_name):
   - если file_name не указан (null), выбрать файл из каталога [PathAFN]
   - инициализировать при необходимости СКАД "Сигнатура"
   - снять ЭЦП
   - распаковать ARJ в PathTEMP
     юзать arj32.exe
   - расшифровать *.VRB
   - unzip *.vrb в .xml
   - снять ЭЦП с xml
   - скопировать в PathINV
   - создать каталог IN\YYYYMMDD и скопировать файлы туда

 ProcessPTK():
  - доп.форма с выбором диапазона дат (м.б. control)
    по умолчанию тек.даты
  - прочитать конфиг ПТК [PathPTK]
  - открыть базу MSAccess по DSN, указанному в ptk.ini
  - Найти в базе посылки mz
  - вывести их в GridView
  - Обраотка посылок:
    - распаковать посылку MZ (через cabarc)
    - скопировать AFN...ARJ в PathAFN
    - вызвать ProcessAFN(file_name)
